# SEO-гайд проекта (Гостевой дом на Зелёной, 26 · Кучугуры)

Документ описывает: **(1)** как пересобрать сайт вручную, **(2)** что реализовано по SEO и где это в коде, **(3)** что и где менять руками в админке, **(4)** разовую настройку внешних сервисов, **(5)** как проверить результат.

Готовые SEO-тексты (заголовки, описания, ключи) лежат в `logs.md` → раздел **«🟢 SEO ДЛЯ САЙТА»**.

---

## 1. Пересборка и запуск (копипаст-команды)

Выполнять **на сервере**, в каталоге проекта (где лежит `docker-compose.yml`).

```bash
# 1. Забрать свежий код на сервер (если код доставляется через git)
git pull

# 2. Пересобрать образ приложения без кэша
docker compose build --no-cache

# 3. Поднять контейнеры
docker compose up -d

# 4. ВАЖНО (только для этого обновления): синхронизировать схему БД —
#    добавит новые колонки seoTitle / seoDescription в таблицу номеров.
docker compose exec app ./node_modules/.bin/prisma db push

# 5. Перезапустить приложение, чтобы оно увидело новые колонки
docker compose restart app
```

### Почему нужен шаг 4
В этом обновлении добавлены два новых поля в базу (`seoTitle`, `seoDescription` у номеров).
Контейнер при старте выполняет `prisma migrate deploy`, но в проекте нет папки миграций,
поэтому колонки сами не создадутся. `prisma db push` синхронизирует схему с базой.

- Команда **безопасна**: новые колонки nullable (необязательные), существующие данные не трогаются и не удаляются.
- Шаг 4 нужен **один раз** — именно для этого обновления. Будущие пересборки **без изменений схемы** делаются только шагами 1–3.
- Если шаг 4 пропустить — страницы номеров будут падать с ошибкой про отсутствующую колонку.

### Проверить, что всё поднялось
```bash
docker compose ps
docker compose logs -f app   # выход — Ctrl+C
```

## 2. Что реализовано по SEO (и где в коде)

| # | Что | Где в коде | Нужно ли трогать руками |
|---|-----|------------|--------------------------|
| 1 | **Отдельные SEO-поля главной** (Title + Description, отдельно от текста на сайте) | `src/app/(public)/page.tsx` → `generateMetadata`, настройки `seo_title` / `seo_description` | Заполнить в админке |
| 2 | **Отдельные SEO-поля каждого номера** (Title + Description) | поля `seoTitle`/`seoDescription` в `prisma/schema.prisma`; вывод — `src/app/(public)/rooms/[id]/page.tsx` | Заполнить в админке |
| 3 | **Мета страницы «Номера»** (`/rooms`) | `src/app/(public)/rooms/page.tsx` → `metadata` | Нет (зашито) |
| 4 | **Авто alt-теги фото** с локацией «Кучугуры» | `src/lib/seo.ts` → `buildRoomImageAlt`; используется в `RoomGallery`, `RoomImageCarousel` | Нет (работает само для всех фото) |
| 5 | **FAQ-блок** (вопросы-ответы) на странице номеров | `src/components/seo/FaqSection.tsx`, тексты в `src/lib/seo.ts` → `FAQ_ITEMS` | Тексты при желании |
| 6 | **Микроразметка Schema.org** (звёзды/цена/FAQ в выдаче) | `src/lib/seo.ts` (билдеры) + `src/components/seo/JsonLd.tsx`; вставлена на главной, странице номера, странице номеров | Нет (зашито) |
| 7 | **sitemap.xml** (карта сайта, включает все номера) | `src/app/sitemap.ts` | Нет |
| 8 | **robots.txt** | `src/app/robots.ts` | Нет |
| 9 | **Яндекс.Метрика** (поведенческие факторы, вебвизор) — только на публичных страницах, не в админке | `src/components/analytics/YandexMetrika.tsx` (счётчик), хиты при SPA-навигации — `Tracker.tsx`; ID из env `NEXT_PUBLIC_YANDEX_METRIKA_ID` (дефолт `109741998`) | Нет |
| 10 | **Иконка вкладки (favicon)** — по конвенции Next из `src/app/` | `src/app/favicon.ico`, `src/app/icon.png`, `src/app/apple-icon.png` | Нет |

> **Фикс (2026-06-09):** `getSiteUrl()` теперь дефолтит на `https://azov.mak-o.ru` (раньше — `localhost`), поэтому robots/sitemap/микроразметка показывают боевой домен даже если `NEXT_PUBLIC_SITE_URL` не задан в `.env`. Иконка перенесена в `src/app/` (раньше лежала в `/images/icons/` и не подхватывалась браузером). Эти изменения — правки кода → деплой шагами 1–3 (без `db push`).

### Принцип «сайт отдельно, поиск отдельно»
- Текст, который видит **посетитель** на сайте (подзаголовок главной, краткое описание номера) — остаётся как есть.
- Текст, который видит **поисковик** в выдаче (SEO-заголовок и SEO-описание) — задаётся отдельными полями.
- **Если SEO-поле пустое** — берётся обычный текст (как было раньше). Ничего не сломается, если их не заполнить.

### Где править локацию (если когда-нибудь сменится посёлок)
Файл `src/lib/seo.ts`, вверху:
```ts
export const SITE_LOCALITY = 'Кучугуры'
export const SITE_REGION = 'Краснодарский край'
export const SITE_STREET = 'ул. Зелёная, 26'
```
Поменял здесь — обновится в alt-тегах, микроразметке и FAQ. После правки — пересборка (шаги 1–3).

## 3. Что менять руками в админке (после деплоя)

Все готовые тексты — в `logs.md`, раздел «🟢 SEO ДЛЯ САЙТА».

### 3.1. Главная страница и общие настройки
**Админка → Настройки → «Информация о сайте»:**

| Поле в админке | Что вставить | Источник в logs.md |
|---|---|---|
| Название сайта | `Отдых в Кучугурах у моря` | разд. 2 |
| Адрес | `Краснодарский край, п. Кучугуры, ул. Зелёная, 26` | разд. 2 |
| Подзаголовок главной | (текст подзаголовка, виден на сайте) | твой текущий |
| **SEO-заголовок** (новое поле) | Title для поисковика, ≤60 симв. | разд. 2 |
| **SEO-описание** (новое поле) | Description для поисковика, ≤160 симв. | разд. 2 |
| Телефон, время заезда/выезда | реальные данные | — |

Нажать **«Сохранить»**.

### 3.2. Каждый номер
**Админка → Номера → «карандаш» (редактировать) у номера:**

| Поле в админке | Что вставить | Источник в logs.md |
|---|---|---|
| Название | `room.name` для номера | разд. 4 |
| Короткое описание | краткое (видно на сайте) | версия «с атмосферой» |
| Полное описание | полное (видно на сайте) | версия «с атмосферой» |
| **SEO-заголовок** (новый блок «SEO номера») | Title для выдачи | разд. 4 |
| **SEO-описание** (новый блок «SEO номера») | Description, ≤160 симв. | разд. 4 |

Повторить для всех номеров (1, 2, 3, 4, 5, 7). Нажать **«Сохранить»**.

> Правило: SEO-поля можно заполнять постепенно. Пустое поле = берётся обычное название/краткое описание.

## 4. Разовая внешняя настройка (бесплатный трафик)

Делается один раз, не в коде:

1. **Яндекс.Вебмастер** (`webmaster.yandex.ru`) — добавить сайт `https://azov.mak-o.ru`, подтвердить права, добавить карту сайта `https://azov.mak-o.ru/sitemap.xml`.
   - **Регион сайта** (КРИТИЧНО для локальной выдачи): Вебмастер → *Информация о сайте → Региональность* → задать **Кучугуры / Темрюкский район / Краснодарский край**. Без региона Яндекс не покажет сайт по запросам «снять жильё в Кучугурах».
   - **Переобход страниц**: Вебмастер → *Индексирование → Переобход страниц* — отправить главную и `/rooms` после деплоя.
2. **Яндекс.Метрика** (`metrika.yandex.ru`) — счётчик `109741998` уже встроен в код. После деплоя: проверить в Метрике, что счётчик «зелёный» (получает данные); включить **Вебвизор** (уже в конфиге) и связать Метрику с Вебмастером (в Вебмастере → *Настройки → Метрика*) — Яндекс будет учитывать поведенческие факторы.
3. **Google Search Console** (`search.google.com/search-console`) — добавить сайт и sitemap.
4. **Яндекс.Бизнес** (`yandex.ru/sprav`) — создать карточку организации (гостевой дом в Кучугурах, адрес, фото, телефон, часы). Даёт трафик с Яндекс.Карт и поиска. Привязать тот же адрес, что на сайте.

> Когда дойдёшь до этого шага — попроси, распишу по кнопкам.

## 5. Как проверить, что SEO работает

После деплоя:

- **sitemap:** открой `https://azov.mak-o.ru/sitemap.xml` — должен показать список страниц и все номера.
- **robots:** открой `https://azov.mak-o.ru/robots.txt` — должен показать правила и ссылку на sitemap.
- **Микроразметка:** вставь адрес номера (напр. `https://azov.mak-o.ru/rooms/<slug>`) в валидатор Яндекса: `webmaster.yandex.ru/tools/microtest/` — должен распознать `HotelRoom`, цену, хлебные крошки. Для FAQ — проверь `https://azov.mak-o.ru/rooms`.
- **Мета-теги:** правый клик по странице → «Просмотр кода» → найди `<title>` и `<meta name="description">` — должны совпадать с тем, что вписал в админке.

## Краткая шпаргалка «обновил тексты — что делать»

- **Поменял только тексты в админке** → ничего пересобирать не нужно, сразу применяется.
- **Поменял код (этот репозиторий)** → шаги 1–3 из раздела 1.
- **Поменял схему БД (`schema.prisma`)** → шаги 1–5 (с `db push`).
